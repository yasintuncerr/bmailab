# 🌐 Ağ ve Yönlendirme Altyapısı Dokümantasyonu

## 📌 Topoloji Özeti

Bu mimaride **core-infra** sunucusu, dış ağ (üniversite ağı) ile iç ağ (compute cluster) arasında bir **NAT Gateway** olarak konumlanır.

- Dış Ağ (WAN): `<WAN_NETWORK_CIDR>`
- İç Ağ (LAN - Cluster): `<LAN_NETWORK_CIDR>`
- Gateway (core-infra):
  - WAN Interface → `<WAN_IFACE>`
  - LAN Interface → `<LAN_IFACE>`

Compute node’lar fiziksel olarak bir switch üzerinden **LAN interface’e bağlıdır** ve tüm dış erişimlerini gateway üzerinden gerçekleştirir.

---

## 🧩 Genel Mimari

```text
[ Üniversite Ağı / Internet ]
              |
        (<WAN_IFACE>)
        [ core-infra ]
        (<LAN_IFACE>)
              |
           [ Switch ]
        /      |      \
 [ Node-1 ] [ Node-2 ] [ Node-N ]
```

---

## 🖥️ 1. Core-Infra (Gateway) Kurulumu

### 🔍 1.1 Interface Tespiti (ÖNEMLİ)

Kuruluma başlamadan önce interface isimlerini doğrula:

```bash
ip -br addr
```

Bulman gerekenler:

- `<WAN_IFACE>` → internet çıkan interface (IP genelde 172.x / DHCP / üniversite ağı)
- `<LAN_IFACE>` → switch’e bağlı interface

---

### ⚙️ 1.2 Netplan Konfigürasyonu

Dosya:

```bash
sudo nano /etc/netplan/01-gateway.yaml
```

Şablon:

```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    <WAN_IFACE>:
      addresses:
        - <WAN_IP>/<WAN_CIDR>
      routes:
        - to: default
          via: <WAN_GATEWAY>
      nameservers:
        addresses:
          - <UPSTREAM_DNS_1>
          - <UPSTREAM_DNS_2>

    <LAN_IFACE>:
      addresses:
        - <LAN_GATEWAY_IP>/<LAN_CIDR>
```

### 📌 Parametre Açıklamaları

| Parametre          | Açıklama                            |
| ------------------ | ----------------------------------- |
| `<WAN_IFACE>`      | İnternete çıkan fiziksel interface  |
| `<WAN_IP>`         | Gateway cihazın dış IP adresi       |
| `<WAN_GATEWAY>`    | Üniversite router IP’si             |
| `<LAN_IFACE>`      | Switch’e bağlı interface            |
| `<LAN_GATEWAY_IP>` | İç ağ gateway IP’si (örn: 10.x.x.1) |

---

### ▶️ Uygulama

```bash
sudo chmod 600 /etc/netplan/01-gateway.yaml
sudo netplan apply
```

---

### 🔁 1.3 IP Forwarding

```bash
echo "net.ipv4.ip_forward=1" | sudo tee /etc/sysctl.d/99-ip-forward.conf
sudo sysctl --system
```

Kontrol:

```bash
cat /proc/sys/net/ipv4/ip_forward
```

Beklenen:

```text
1
```

---

### 🌍 1.4 NAT (Internet Çıkışı)

```bash
sudo iptables -t nat -A POSTROUTING -o <WAN_IFACE> -j MASQUERADE
```

Kalıcı yapmak için:

```bash
sudo apt update
sudo apt install iptables-persistent -y
sudo netfilter-persistent save
```

## 💻 2. Compute Node Kurulumu

### 🔍 2.1 Interface Tespiti

```bash
ip -br addr
```

Genelde tek interface olur:

- `<NODE_IFACE>`

---

### ⚙️ 2.2 Netplan Konfigürasyonu

```bash
sudo nano /etc/netplan/01-node.yaml
```

```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    <NODE_IFACE>:
      dhcp4: no
      addresses:
        - <NODE_IP>/<LAN_CIDR>
      routes:
        - to: default
          via: <LAN_GATEWAY_IP>
      nameservers:
        addresses:
          - <LAN_GATEWAY_IP>
      dhcp4-overrides:
        use-dns: false
```

---

### ▶️ Uygulama

```bash
sudo chmod 600 /etc/netplan/01-node.yaml
sudo netplan apply
```

## 🧠 DNS Yapısı

- Tüm node’lar DNS olarak **gateway’i kullanır**
- Gateway üzerinde **Technitium DNS** çalışır
- İç domain: `<LAB_DOMAIN>`

📌 Örnek:

```text
node01.<LAB_DOMAIN> → <NODE_IP>
```

## ⚠️ Yaygın Problemler

### ❌ Internet yok

- NAT kuralı yok
- ip_forward kapalı

### ❌ DNS çalışmıyor

- nameserver yanlış
- Technitium çalışmıyor

### ❌ Node gateway’e ulaşamıyor

- yanlış interface seçilmiş olabilir
- switch bağlantısı hatalı olabilir

---

## 🎯 Tasarım Notları

- Tüm yapı **interface bağımsız** tasarlanmıştır
- Kurulum öncesi **interface ve IP keşfi zorunludur**
- Parametreler ortam bazlı değiştirilebilir
- Mimari **yatay ölçeklenebilir (scale-out)** yapıdadır

---

## 📌 Özet

- core-infra → NAT Gateway + DNS
- compute node → sadece gateway’e bağlı
- tüm trafik → gateway üzerinden çıkar
- DNS → merkezi ve dinamik yönetilir
