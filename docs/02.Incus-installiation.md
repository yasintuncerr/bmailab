# ⚙️ Incus Kurulum Rehberi — Core-Infra Node

## 📌 Genel Bakış

Bu doküman, **core-infra** sunucusunda Incus'un ilk cluster node olarak kurulumunu ve web arayüzü erişimini adım adım açıklar.

Incus, [Zabbly](https://github.com/zabbly/incus) tarafından sağlanan kararlı (stable) repository üzerinden kurulur.

---

## 📦 1. Zabbly Repository Kurulumu

### 🔑 1.1 GPG Anahtarı Doğrulaması

Paketi imzalamak için kullanılan public key'in parmak izini doğrula:

```bash
curl -fsSL https://pkgs.zabbly.com/key.asc | gpg --show-keys --fingerprint
```

`curl` yoksa `wget` ile:

```bash
wget -q -O - https://pkgs.zabbly.com/key.asc | gpg --show-keys --fingerprint
```

Beklenen çıktı:

```
pub   rsa3072 2023-08-23 [SC] [expires: 2030-08-17]
      4EFC 5906 96CB 15B8 7C73  A3AD 82CC 8797 C838 DCFD
uid                      Zabbly Kernel Builds <info@zabbly.com>
sub   rsa3072 2023-08-23 [E] [expires: 2030-08-17]
```

> ⚠️ Parmak izi eşleşmiyorsa kuruluma devam etme.

---

### 🗂️ 1.2 Anahtar Kaydı

```bash
mkdir -p /etc/apt/keyrings/
curl -fsSL https://pkgs.zabbly.com/key.asc -o /etc/apt/keyrings/zabbly.asc
```

---

### 📋 1.3 Repository Ekleme

```bash
sh -c 'cat <<EOF > /etc/apt/sources.list.d/zabbly-incus-stable.sources
Enabled: yes
Types: deb
URIs: https://pkgs.zabbly.com/incus/stable
Suites: $(. /etc/os-release && echo ${VERSION_CODENAME})
Components: main
Architectures: $(dpkg --print-architecture)
Signed-By: /etc/apt/keyrings/zabbly.asc

EOF'
```

## 🔧 3. Disk Hazırlığı (ZFS Pool için)

ZFS pool olarak kullanılacak disk daha önce formatlanmışsa, `incus admin init` sırasında aşağıdaki hata alınabilir:

```
Error: Failed to create storage pool "local": Failed to run: zpool create ...
/dev/<POOL_DISK> contains a filesystem of type 'ext4'
```

Bu durumda diski sıfırla:

```bash
sudo wipefs -a /dev/<POOL_DISK>
```

| Parametre     | Açıklama                                         |
| ------------- | ------------------------------------------------ |
| `<POOL_DISK>` | ZFS pool için ayrılan fiziksel disk (örn: `sdb`) |


## 🖥️ 5. Web Arayüzü Erişimi (incus-ui-canonical)

Web arayüzüne erişim için TLS client sertifikası üretilmesi ve tarayıcıya eklenmesi gerekir.

### 5.1 Sertifika Üretimi

```bash
openssl req -x509 -newkey rsa:4096 -sha256 -days 3650 -nodes \
  -keyout incus-ui.key \
  -out incus-ui.crt \
  -subj "/CN=incus-ui"

openssl pkcs12 -export \
  -out incus-ui.pfx \
  -inkey incus-ui.key \
  -in incus-ui.crt \
  -passout pass:<PFX_PASSWORD>
```

| Parametre        | Açıklama                                         |
| ---------------- | ------------------------------------------------ |
| `<PFX_PASSWORD>` | PFX dosyasına atanacak şifre (boş bırakılabilir) |

---

### 5.2 Sertifikayı Incus'a Tanıt

Üretilen `.crt` dosyasını core-infra'ya kopyaladıktan sonra:

```bash
sudo incus config trust add-certificate incus-ui.crt
```

---

### 5.3 Tarayıcıya PFX Ekle (Firefox)

1. `about:preferences#privacy` adresine git
2. **Certificates → View Certificates → Your Certificates → Import** seçeneğini kullan
3. `incus-ui.pfx` dosyasını ve şifreyi gir

---

### 5.4 Arayüze Erişim

SSH tüneli ile bağlan:

```bash
ssh -L 8443:<CLUSTER_LISTEN_IP>:8443 <SSH_USER>@<HOST_IP>
```

Ardından tarayıcıda aç:

```
https://localhost:8443
```

| Parametre             | Açıklama                           |
| --------------------- | ---------------------------------- |
| `<CLUSTER_LISTEN_IP>` | Incus API'nin dinlediği IP         |
| `<SSH_USER>`          | SSH kullanıcı adı                  |
| `<HOST_IP>`           | core-infra'nın dış (WAN) IP adresi |

## ⚠️ Yaygın Problemler

### ❌ ZFS pool oluşturulamıyor

```
/dev/<DISK> contains a filesystem of type 'ext4'
```

**Çözüm:**

```bash
sudo wipefs -a /dev/<POOL_DISK>
```

---

### ❌ Web UI'ye bağlanılamıyor

- Sertifika tarayıcıya eklenmemiş olabilir
- `incus config trust add-certificate` çalıştırılmamış olabilir
- SSH tüneli aktif değildir

---

### ❌ API dinlemiyor

```bash
incus config set core.https_address <CLUSTER_LISTEN_IP>:8443
```
