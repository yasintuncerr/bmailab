# ⚙️ Incus Kurulum Rehberi

## 📌 Genel Bakış

Bu doküman, Incus cluster'ının adım adım kurulumunu kapsar:

1. **core-infra** — ilk cluster node (bootstrap)
2. **compute node** — cluster'a katılan iş yükü node'u

Incus, [Zabbly](https://github.com/zabbly/incus) tarafından sağlanan kararlı (stable) repository üzerinden kurulur.

---

## ⚠️ Ön Koşullar

- Ubuntu Server 22.04 veya 24.04 kurulu
- Ağ yapılandırması tamamlanmış (bkz. `01.Network&Routing.md`)
- `root` veya `sudo` yetkisi mevcut
- Her iki node birbirine ping atabilir durumda

---

## 📦 1. Zabbly Repository Kurulumu

> Bu adımlar **her node'da** (core-infra ve compute node) ayrı ayrı yapılır.

### 🔑 1.1 GPG Anahtarı Doğrulaması

Paketi imzalamak için kullanılan public key'in parmak izini doğrula:

```bash
curl -fsSL https://pkgs.zabbly.com/key.asc | gpg --show-keys --fingerprint
```

`curl` yoksa `wget` ile:

```bash
wget -q -O - https://pkgs.zabbly.com/key.asc | gpg --show-keys --fingerprint
```

Beklenen çıktı:

```
pub   rsa3072 2023-08-23 [SC] [expires: 2030-08-17]
      4EFC 5906 96CB 15B8 7C73  A3AD 82CC 8797 C838 DCFD
uid                      Zabbly Kernel Builds <info@zabbly.com>
sub   rsa3072 2023-08-23 [E] [expires: 2030-08-17]
```

> ⚠️ Parmak izi eşleşmiyorsa kuruluma devam etme. Key güvenliği tehlikede olabilir.

---

### 🗂️ 1.2 Anahtar Kaydı

```bash
sudo mkdir -p /etc/apt/keyrings/
sudo curl -fsSL https://pkgs.zabbly.com/key.asc -o /etc/apt/keyrings/zabbly.asc
```

Doğrulama:

```bash
ls -la /etc/apt/keyrings/zabbly.asc
```

---

### 📋 1.3 Repository Ekleme

```bash
sudo sh -c 'cat <<EOF > /etc/apt/sources.list.d/zabbly-incus-stable.sources
Enabled: yes
Types: deb
URIs: https://pkgs.zabbly.com/incus/stable
Suites: $(. /etc/os-release && echo ${VERSION_CODENAME})
Components: main
Architectures: $(dpkg --print-architecture)
Signed-By: /etc/apt/keyrings/zabbly.asc

EOF'
```

Repository'nin eklendiğini doğrula:

```bash
cat /etc/apt/sources.list.d/zabbly-incus-stable.sources
```

## 🛠️ 2. Incus Kurulumu

> Bu adım **her node'da** yapılır.

```bash
sudo apt-get update
sudo apt-get install incus incus-ui-canonical -y
```

| Paket                | Açıklama                          |
| -------------------- | --------------------------------- |
| `incus`              | Ana daemon + CLI                  |
| `incus-ui-canonical` | Web arayüzü (LXD UI rebranded)    |
| `incus-client`       | Sadece CLI — uzak sunucu yönetimi |

Kurulum sonrası servis durumunu kontrol et:

```bash
systemctl status incus
```

`Active: active (running)` görülmelidir.

## 🔧 3. Disk Hazırlığı (ZFS Pool için)

> Bu adım yalnızca **ZFS pool için ayrılan diskte** yapılır.

ZFS pool olarak kullanılacak disk önceden formatlanmışsa `incus admin init` sırasında şu hata alınabilir:

```
Error: Failed to create storage pool "local": Failed to run: zpool create ...
/dev/<POOL_DISK> contains a filesystem of type 'ext4'
```

Bu durumda diski sıfırla:

```bash
sudo wipefs -a /dev/<POOL_DISK>
```

Doğrulama:

```bash
sudo blkid /dev/<POOL_DISK>
```

Çıktı boş olmalı — herhangi bir filesystem imzası görünmemeli.

| Parametre     | Açıklama                                         | Örnek  |
| ------------- | ------------------------------------------------ | ------ |
| `<POOL_DISK>` | ZFS pool için ayrılan fiziksel disk              | `sdb`  |

## ⚙️ 4. Cluster Bootstrap (Core-Infra)

> Bu adım **yalnızca core-infra'da** yapılır.

```bash
sudo incus admin init
```

Aşağıdaki sorular ve önerilen cevaplar:

---

### Clustering

```
Would you like to use clustering? (yes/no) [default=no]: yes
```

```
Are you joining an existing cluster? (yes/no) [default=no]: no
```

```
What member name should be used to identify this server in the cluster? [default=...]: <CLUSTER_MEMBER_NAME>
```

| Parametre               | Açıklama                                            | Örnek         |
| ----------------------- | --------------------------------------------------- | ------------- |
| `<CLUSTER_MEMBER_NAME>` | Bu node'un cluster içindeki adı                     | `core-infra`  |

---

### Storage Pool

```
Setup a new local storage pool? (yes/no) [default=yes]: yes
```

```
Would you like to create a new ZFS dataset under rpool/incus? (yes/no): no
```

```
Would you like to use an existing empty block device? (yes/no) [default=no]: yes
```

```
Create a new ZFS pool? (yes/no) [default=yes]: yes
```

```
Which block device or dataset should be used? (e.g., /dev/sdb): /dev/<POOL_DISK>
```

---

### Ağ

```
Would you like to create a new local network bridge? (yes/no) [default=yes]: yes
```

```
What should the new bridge be called? [default=incusbr0]: <INCUS_BRIDGE_NAME>
```

```
What IPv4 address should be used? [default=auto]: <INCUS_BRIDGE_IP>/<INCUS_BRIDGE_CIDR>
```

| Parametre             | Açıklama                                       | Örnek          |
| --------------------- | ---------------------------------------------- | -------------- |
| `<INCUS_BRIDGE_NAME>` | Köprü arayüz adı                               | `incusbr0`     |
| `<INCUS_BRIDGE_IP>`   | Köprünün gateway IP'si (container default gw)  | `10.14.21.1`   |
| `<INCUS_BRIDGE_CIDR>` | Alt ağ maskesi                                 | `24`           |

> 📌 Bu subnet, LAN subnet'i ile çakışmamalıdır. LAN `10.10.0.0/24` ise Incus bridge için `10.14.21.0/24` gibi farklı bir range kullan.

---

### Uzaktan Erişim (Web UI + API)

```
Would you like the server to be available over the network? (yes/no) [default=no]: yes
```

```
What IP address or DNS name should be used? [default=all]: <CLUSTER_LISTEN_IP>
```

| Parametre             | Açıklama                                              | Örnek          |
| --------------------- | ----------------------------------------------------- | -------------- |
| `<CLUSTER_LISTEN_IP>` | Cluster API'nin dinleyeceği IP — diğer node'lar buna bağlanır | `10.10.0.1` |

---

### Diğer

```
Would you like stale cached images to be updated automatically? (yes/no) [default=yes]: yes
Would you like a YAML "init" preseed to be printed? (yes/no) [default=no]: no
```

---

### Bootstrap Doğrulaması

```bash
# Cluster durumu
incus cluster list

# Ağ bridge'i
incus network list

# Storage pool
incus storage list

# Boş instance listesi (henüz instance yok)
incus list
```

## 🖥️ 5. Compute Node'u Cluster'a Ekleme

### 5.1 Core-Infra'da Join Token Üret

```bash
incus cluster add <COMPUTE_NODE_NAME>
```

Bu komut tek kullanımlık bir join token üretir. Token'ı kopyala.

| Parametre              | Açıklama                           | Örnek            |
| ---------------------- | ---------------------------------- | ---------------- |
| `<COMPUTE_NODE_NAME>`  | Cluster'daki görünür ad            | `compute-01`     |

---

### 5.2 Compute Node'da Init Çalıştır

Compute node'a SSH ile bağlan ve:

```bash
sudo incus admin init
```

---

### Clustering

```
Would you like to use clustering? (yes/no) [default=no]: yes
```

```
Are you joining an existing cluster? (yes/no) [default=no]: yes
```

```
Please provide join token: <YUKARIDA_KOPYALANAN_TOKEN>
```

Token girildikten sonra sihirbaz otomatik olarak core-infra'ya bağlanır ve cluster bilgilerini çeker.

---

### Storage Pool (Compute Node)

```
Would you like to create a new local storage pool? (yes/no): yes
```

```
Would you like to use an existing empty block device? (yes/no): yes
```

```
Which block device or dataset should be used? (e.g., /dev/sdb): /dev/<COMPUTE_POOL_DISK>
```

---

### Network (Compute Node)

```
Would you like to create a new local network bridge? (yes/no): no
```

> 📌 Compute node kendi bridge oluşturmaz; zaten core-infra'da oluşturulan bridge'i kullanır.

---

### 5.3 Join Doğrulaması

Core-infra'da:

```bash
incus cluster list
```

Beklenen çıktı (her iki node da `ONLINE` görünmeli):

```
+-------------+-------------------+------------------+--------+-------------+
| NAME        | URL               | ROLES            | STATUS | MESSAGE     |
+-------------+-------------------+------------------+--------+-------------+
| core-infra  | https://10.10.0.1 | database-leader  | ONLINE | Full quorum |
| compute-01  | https://10.10.0.2 | database         | ONLINE | Full quorum |
+-------------+-------------------+------------------+--------+-------------+
```

## 🏷️ 6. `dns-enabled` Profili Oluşturma

`incus-dns-sync` servisi, DNS'e kaydedilecek instance'ları bu profile bakarak belirler. Profil içeriği boştur — yalnızca bir etiket işlevi görür.

```bash
incus profile create dns-enabled
```

Doğrulama:

```bash
incus profile list
```

## 🖥️ 7. Web Arayüzü Erişimi (incus-ui-canonical)

Web arayüzüne erişim için TLS client sertifikası üretilmesi ve tarayıcıya eklenmesi gerekir.

### 7.1 Sertifika Üretimi

```bash
openssl req -x509 -newkey rsa:4096 -sha256 -days 3650 -nodes \
  -keyout incus-ui.key \
  -out incus-ui.crt \
  -subj "/CN=incus-ui"

openssl pkcs12 -export \
  -out incus-ui.pfx \
  -inkey incus-ui.key \
  -in incus-ui.crt \
  -passout pass:<PFX_PASSWORD>
```

| Parametre        | Açıklama                                         |
| ---------------- | ------------------------------------------------ |
| `<PFX_PASSWORD>` | PFX dosyasına atanacak şifre (boş bırakılabilir) |

---

### 7.2 Sertifikayı Incus'a Tanıt

```bash
sudo incus config trust add-certificate incus-ui.crt
```

Doğrulama:

```bash
incus config trust list
```

`incus-ui` CN'ine sahip bir sertifika görünmelidir.

---

### 7.3 Tarayıcıya PFX Ekle (Firefox)

1. `about:preferences#privacy` adresine git
2. **Certificates → View Certificates → Your Certificates → Import** seçeneğini kullan
3. `incus-ui.pfx` dosyasını ve şifreyi gir

---

### 7.4 Arayüze Erişim

SSH tüneli ile bağlan:

```bash
ssh -L 8443:<CLUSTER_LISTEN_IP>:8443 <SSH_USER>@<HOST_WAN_IP>
```

Ardından tarayıcıda aç:

```
https://localhost:8443
```

| Parametre             | Açıklama                            |
| --------------------- | ----------------------------------- |
| `<CLUSTER_LISTEN_IP>` | Incus API'nin dinlediği IP          |
| `<SSH_USER>`          | SSH kullanıcı adı                   |
| `<HOST_WAN_IP>`       | core-infra'nın dış (WAN) IP adresi |

## ⚠️ Yaygın Problemler

### ❌ ZFS pool oluşturulamıyor

```
/dev/<DISK> contains a filesystem of type 'ext4'
```

**Çözüm:**

```bash
sudo wipefs -a /dev/<POOL_DISK>
```

---

### ❌ Compute node join token geçersiz

Token tek kullanımlıktır ve genellikle 3 saat geçerlidir. Süresi dolduysa:

```bash
# Core-infra'da yeni token üret
incus cluster add <COMPUTE_NODE_NAME>
```

---

### ❌ Web UI'ye bağlanılamıyor

1. Sertifika tarayıcıya eklenmemiş olabilir
2. `incus config trust add-certificate` çalıştırılmamış olabilir
3. SSH tüneli aktif değildir

---

### ❌ API dinlemiyor

```bash
incus config set core.https_address <CLUSTER_LISTEN_IP>:8443
```
