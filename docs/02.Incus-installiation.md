# ⚙️ Incus Kurulum Rehberi

## 📌 Genel Bakış

Bu doküman, Incus cluster'ının adım adım kurulumunu kapsar:

1. **core-infra** — ilk cluster node (bootstrap)
2. **compute node** — cluster'a katılan iş yükü node'u

Incus, [Zabbly](https://github.com/zabbly/incus) tarafından sağlanan kararlı (stable) repository üzerinden kurulur.

---

## ⚠️ Ön Koşullar

- Ubuntu Server 22.04 veya 24.04 kurulu
- Ağ yapılandırması tamamlanmış (bkz. `01.Network&Routing.md`)
- `root` veya `sudo` yetkisi mevcut
- Her iki node birbirine ping atabilir durumda

---

## 📦 1. Zabbly Repository Kurulumu

> Bu adımlar **her node'da** (core-infra ve compute node) ayrı ayrı yapılır.

### 🔑 1.1 GPG Anahtarı Doğrulaması

Paketi imzalamak için kullanılan public key'in parmak izini doğrula:

```bash
curl -fsSL https://pkgs.zabbly.com/key.asc | gpg --show-keys --fingerprint
```

`curl` yoksa `wget` ile:

```bash
wget -q -O - https://pkgs.zabbly.com/key.asc | gpg --show-keys --fingerprint
```

Beklenen çıktı:

```
pub   rsa3072 2023-08-23 [SC] [expires: 2030-08-17]
      4EFC 5906 96CB 15B8 7C73  A3AD 82CC 8797 C838 DCFD
uid                      Zabbly Kernel Builds <info@zabbly.com>
sub   rsa3072 2023-08-23 [E] [expires: 2030-08-17]
```

> ⚠️ Parmak izi eşleşmiyorsa kuruluma devam etme. Key güvenliği tehlikede olabilir.

---

### 🗂️ 1.2 Anahtar Kaydı

```bash
sudo mkdir -p /etc/apt/keyrings/
sudo curl -fsSL https://pkgs.zabbly.com/key.asc -o /etc/apt/keyrings/zabbly.asc
```

Doğrulama:

```bash
ls -la /etc/apt/keyrings/zabbly.asc
```

---

### 📋 1.3 Repository Ekleme

```bash
sudo sh -c 'cat <<EOF > /etc/apt/sources.list.d/zabbly-incus-stable.sources
Enabled: yes
Types: deb
URIs: https://pkgs.zabbly.com/incus/stable
Suites: $(. /etc/os-release && echo ${VERSION_CODENAME})
Components: main
Architectures: $(dpkg --print-architecture)
Signed-By: /etc/apt/keyrings/zabbly.asc

EOF'
```

Repository'nin eklendiğini doğrula:

```bash
cat /etc/apt/sources.list.d/zabbly-incus-stable.sources
```

## 🛠️ 2. Incus Kurulumu

> Bu adım **her node'da** yapılır.

```bash
sudo apt-get update
sudo apt-get install incus incus-ui-canonical -y
```

| Paket                | Açıklama                          |
| -------------------- | --------------------------------- |
| `incus`              | Ana daemon + CLI                  |
| `incus-ui-canonical` | Web arayüzü (LXD UI rebranded)    |
| `incus-client`       | Sadece CLI — uzak sunucu yönetimi |

Kurulum sonrası servis durumunu kontrol et:

```bash
systemctl status incus
```

`Active: active (running)` görülmelidir.

## 🔧 3. Disk Hazırlığı (ZFS Pool için)

> Bu adım yalnızca **ZFS pool için ayrılan diskte** yapılır.

ZFS pool olarak kullanılacak disk önceden formatlanmışsa `incus admin init` sırasında şu hata alınabilir:

```
Error: Failed to create storage pool "local": Failed to run: zpool create ...
/dev/<POOL_DISK> contains a filesystem of type 'ext4'
```

Bu durumda diski sıfırla:

```bash
sudo wipefs -a /dev/<POOL_DISK>
```

Doğrulama:

```bash
sudo blkid /dev/<POOL_DISK>
```

Çıktı boş olmalı — herhangi bir filesystem imzası görünmemeli.

| Parametre     | Açıklama                                         | Örnek  |
| ------------- | ------------------------------------------------ | ------ |
| `<POOL_DISK>` | ZFS pool için ayrılan fiziksel disk              | `sdb`  |

## ⚙️ 4. Cluster Bootstrap (Core-Infra)

> Bu adım **yalnızca core-infra'da** yapılır.

```bash
sudo incus admin init
```

Aşağıdaki sorular ve cevaplar bu sıralama ile verilmiştir:

```text
Are you joining an existing cluster? (yes/no) [default=no]: no
```

## 🕸️ 5. Manuel Ağ Yapılandırması (incusbr0)

`incus admin init` sırasında bridge oluşturma adımı atlandığı için, bridge manuel olarak oluşturulur. Bu adım **yalnızca core-infra'da** yapılır.

```bash
incus network create <INCUS_BRIDGE_NAME> \
    ipv4.address=<INCUS_BRIDGE_IP>/<INCUS_BRIDGE_CIDR> \
    ipv4.nat=true \
    ipv6.address=none
```

| Parametre             | Açıklama                                       | Örnek          |
| --------------------- | ---------------------------------------------- | -------------- |
| `<INCUS_BRIDGE_NAME>` | Köprü arayüz adı                               | `incusbr0`     |
| `<INCUS_BRIDGE_IP>`   | Köprünün gateway IP'si (container default gw)  | `10.11.0.1`    |
| `<INCUS_BRIDGE_CIDR>` | Alt ağ maskesi                                 | `24`           |

## 🖥️ 6. Compute Node'u Cluster'a Ekleme

### 6.1 Core-Infra'da Join Token Üret

```bash
incus cluster add <COMPUTE_NODE_NAME>
```

Bu komut tek kullanımlık bir join token üretir. Token'ı kopyala.

| Parametre              | Açıklama                           | Örnek            |
| ---------------------- | ---------------------------------- | ---------------- |
| `<COMPUTE_NODE_NAME>`  | Cluster'daki görünür ad            | `compute-01`     |

---

### 6.2 Compute Node'da Init Çalıştır

Compute node'a SSH ile bağlan ve:

```bash
sudo incus admin init
```


Token girildikten sonra sihirbaz otomatik olarak core-infra'ya bağlanır ve cluster bilgilerini çeker.

## 🖥️ 8. Web Arayüzü Erişimi (Incus UI + Authentik OIDC)

Incus web arayüzüne erişim için TLS istemci sertifikaları kullanmak yerine Authentik üzerinden Tek Oturum Açma (SSO/OIDC) entegrasyonu yapılandırılır.

```text
[Tarayıcı] → https://incus.bmlab.uludag.edu.tr
                    │
              Traefik (file provider)
                    │
              https://10.10.0.1:8443 (Incus)
                    │
              "Login with SSO" tıklanır
                    │
              Authentik OIDC → login
                    │
              Incus UI erişimi
```

### 8.1 Authentik: OIDC Provider Oluşturma

Authentik Admin UI'da: **Applications → Providers → Create → OAuth2/OpenID Provider**

| Alan | Değer |
|------|-------|
| Name | `incus-oidc` |
| Authorization flow | `default-provider-authorization-explicit-consent` |
| Client type | `Public` (Önemli: Confidential seçilirse Incus `invalid_client` hatası verir) |
| Redirect URIs | `https://incus.bmlab.uludag.edu.tr/oidc/callback` |
| Signing Key | `authentik Self-signed Certificate` |

> ⚠️ Kaydettikten sonra otomatik üretilen **Client ID** değerini not alın.

### 8.2 Authentik: Application Oluşturma

Authentik Admin UI'da: **Applications → Applications → Create**

| Alan | Değer |
|------|-------|
| Name | `Incus UI` |
| Slug | `incus-ui` |
| Provider | `incus-oidc` |
| Policy engine mode | `any` |
| Group | `incus-admins` |

### 8.3 Authentik: Group ve Policy Binding

1. **Group Oluşturma:** Directory → Groups → Create (`Name: incus-admins`)
2. **Kullanıcı Ekleme:** Directory → Users → `<kullanıcı>` → Groups → Add → `incus-admins`
3. **Policy Binding:** Applications → `incus-ui` → Policy/Group/User Bindings → Create Binding

| Alan | Değer |
|------|-------|
| Type | Group |
| Group | `incus-admins` |
| Enabled | ✅ |
| Negate result | ❌ |
| Order | `0` |
| Timeout | `30` |
| Failure result | `Pass` |

### 8.4 Incus OIDC Yapılandırması

Core-infra sunucusunda OIDC ayarlarını uygulayın:

```bash
# OIDC issuer — Authentik application slug'ına göre
sudo incus config set oidc.issuer https://auth.bmlab.uludag.edu.tr/application/o/incus-ui/

# Client ID — Authentik'ten kopyaladığınız değer
sudo incus config set oidc.client.id <CLIENT_ID>

# Doğrula
sudo incus config show | grep oidc
```

> 📌 Incus 6.23 ve üzerinde PKCE kullanıldığı için `oidc.client.secret` gerekmez.

### 8.5 Self-Signed Sertifika Güveni

Incus, Authentik'in Traefik üzerinden sunduğu self-signed sertifikaya varsayılan olarak güvenmez. Bu sertifikayı core-infra sistemine tanıtmak gerekir:

```bash
# Sertifikayı Alpine'dan al
sudo incus exec infra-gateway -- cat /opt/infra/config/certs/cert.pem | \
  sudo tee /usr/local/share/ca-certificates/authentik-self.crt

# CA deposunu güncelle
sudo update-ca-certificates

# Incus'u yeniden başlat
sudo systemctl restart incus
```

### 8.6 Traefik: Incus UI File Provider

Incus bir Docker container'ı olmadığı için Traefik'e label ile değil, dosya sağlayıcı (file provider) ile tanıtılır:

```bash
incus exec infra-gateway -- ash -c "cat << 'EOF' > /opt/infra/config/certs/incus.yml
http:
  routers:
    incus:
      rule: \"Host(\`incus.bmlab.uludag.edu.tr\`)\"
      entryPoints:
        - websecure
      tls: {}
      service: incus

  services:
    incus:
      loadBalancer:
        passHostHeader: true
        serversTransport: incus-transport
        servers:
          - url: \"https://10.10.0.1:8443\"

  serversTransports:
    incus-transport:
      insecureSkipVerify: true
EOF"
```

> 📌 Traefik, `/certs` klasörünü `--providers.file.watch=true` ile izlediği için yeniden başlatmaya gerek kalmadan bu yapılandırmayı otomatik olarak devreye alır.

### 8.7 DNS ve Hosts Ayarları

**Technitium DNS'te:**
`incus.bmlab.uludag.edu.tr` için `infra.bmlab.uludag.edu.tr` adresine bir CNAME kaydı oluşturun.

**Mac / İstemci `/etc/hosts`'ta (DNS tam oturana kadar):**
```text
172.18.107.62  incus.bmlab.uludag.edu.tr
```

### 8.8 Doğrulama ve Kullanım

```bash
# Traefik router'ı görüyor mu? (İstemciden test)
curl -sk https://traefik.bmlab.uludag.edu.tr/api/http/routers | grep incus

# Incus UI yanıt veriyor mu?
curl -k https://incus.bmlab.uludag.edu.tr
```

Tarayıcınızdan `https://incus.bmlab.uludag.edu.tr` adresine gidin, **"Login with SSO"** butonuna tıklayın ve Authentik üzerinden giriş yaparak Incus UI'a erişim sağlayın.

## ⚠️ Yaygın Problemler

### ❌ ZFS pool oluşturulamıyor

```
/dev/<DISK> contains a filesystem of type 'ext4'
```

**Çözüm:**

```bash
sudo wipefs -a /dev/<POOL_DISK>
```

---

### ❌ Compute node join token geçersiz

Token tek kullanımlıktır ve genellikle 3 saat geçerlidir. Süresi dolduysa:

```bash
# Core-infra'da yeni token üret
incus cluster add <COMPUTE_NODE_NAME>
```

---

### ❌ OIDC / Web UI'ye bağlanılamıyor veya Hata Alınıyor

| Hata | Neden | Çözüm |
|------|-------|-------|
| `Redirect URI Error` | Authentik'te redirect URI eksik/yanlış | Provider'a `https://incus.bmlab.uludag.edu.tr/oidc/callback` ekle |
| `invalid_client` | Client type Confidential | Provider'ı **Public** yap |
| `x509: certificate signed by unknown authority` | Incus self-signed sertifikaya güvenmiyor | `update-ca-certificates` ile sertifikayı sisteme ekle |
| `OpenID Provider Configuration Discovery has failed` | Sertifika güven sorunu | Adım 8.5'i tekrar uygula |

---

### ❌ API dinlemiyor

```bash
incus config set core.https_address <CLUSTER_LISTEN_IP>:8443
```
