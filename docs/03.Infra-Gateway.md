# 🛡️ infra-gateway Kurulum Rehberi

## 📌 Genel Bakış

`infra-gateway`, Incus cluster'ının servis katmanını barındıran özel bir container'dır. Alpine Linux tabanlı bu container içinde Docker çalıştırılır; DNS, proxy ve kimlik doğrulama servisleri Docker Compose ile yönetilir.

| Servis     | Image                   | Görev                       |
| ---------- | ----------------------- | --------------------------- |
| Technitium | `technitium/dns-server` | İç DNS yönetimi ve API      |
| Traefik    | `traefik:v3.0`          | HTTP/SSH reverse proxy      |
| Authentik  | `goauthentik/server`    | SSO kimlik doğrulama        |

### Ağ Topolojisi

```text
[Uzak Makine (Mac)]
        │
        │ SSH / HTTPS
        ▼
172.18.107.62 — core-infra (WAN)        [enp0s25]
        │
10.10.0.1/24  — core-infra (LAN)        [enp6s0f1]
        │
10.11.0.1/24  — Incus Bridge            [incusbr0]
        │
10.11.0.2     — Alpine Linux (Docker Host)
        ├── Technitium DNS   :53 / :5380
        ├── Traefik          :80 / :443 / :8080
        └── Authentik        :9000
```

## 🖥️ 1. Container Oluşturma

### 1.1 Alpine Instance Başlatma

Docker çalıştırabilmek için `security.nesting=true` doğrudan `launch` sırasında verilir; sonradan `restart` gerekmez:

```bash
incus launch images:alpine/3.20 <GATEWAY_INSTANCE_NAME> \
  --profile default \
  -c security.nesting=true
```

| Parametre                 | Açıklama                             | Örnek           |
| ------------------------- | ------------------------------------ | --------------- |
| `<GATEWAY_INSTANCE_NAME>` | Container adı                        | `infra-gateway` |

> 📌 `security.nesting=true` olmadan Docker daemon container içinde başlamaz — `iptables` ve `overlay` filesystem yetkisi bu ayarla gelir.

Container ayağa kalktığını doğrula:

```bash
incus list <GATEWAY_INSTANCE_NAME>
```

### 1.2 Statik IP Ataması

Container'a sabit IP atamak için:

```bash
incus config device override <GATEWAY_INSTANCE_NAME> eth0 ipv4.address=<GATEWAY_IP>
incus restart <GATEWAY_INSTANCE_NAME>
```

`IPV4` sütununda `<GATEWAY_IP>` görünmelidir (`10.11.0.2`).

## 🔥 2. Ağ ve Yönlendirme

### 2.1 Host UFW Yapılandırması

Docker ve Incus container trafiğinin geçebilmesi için host üzerindeki UFW'nin ilgili arayüzü engellemediğinden emin olunmalıdır.

```bash
sudo ufw allow in on incusbr0
sudo ufw allow out on incusbr0
sudo ufw reload
```

### 2.2 Core-infra: Port Forwarding (iptables)

WAN'dan gelen HTTP (80) ve HTTPS (443) trafiğini doğrudan Alpine instance'ına (Traefik) yönlendirmek için core-infra makinesinde DNAT ayarlanmalıdır:

```bash
IFACE=enp0s25

# HTTP ve HTTPS için DNAT
sudo iptables -t nat -A PREROUTING -i $IFACE -p tcp --dport 80 \
  -j DNAT --to-destination 10.11.0.2:80

sudo iptables -t nat -A PREROUTING -i $IFACE -p tcp --dport 443 \
  -j DNAT --to-destination 10.11.0.2:443

# Çıkış trafiği için MASQUERADE
sudo iptables -t nat -A POSTROUTING -d 10.11.0.2 -j MASQUERADE

sudo netfilter-persistent save
```

> 📌 `ip_forward` açık olduğu doğrulanmalıdır: `cat /proc/sys/net/ipv4/ip_forward` → `1` dönmeli.

## 📁 4. Dizin Yapısı ve SSL Sertifikası

### 4.1 Dizin Yapısı

Container içinde gerekli dizinleri oluştur (Authentik için `1000:1000` izinleri gereklidir):

```bash
# Technitium ve Sertifikalar
mkdir -p /opt/infra/config/dns
mkdir -p /opt/infra/config/certs

# Authentik (DB ve Media/Certs/Templates)
mkdir -p /opt/infra/data/authentik/media
mkdir -p /opt/infra/data/authentik/templates
mkdir -p /opt/infra/data/authentik/certs
mkdir -p /opt/infra/data/postgresql

# Authentik izinlerini düzelt
chown -R 1000:1000 /opt/infra/data/authentik
chmod -R 755 /opt/infra/data/authentik
```

### 4.2 Self-Signed SSL Sertifikası

Domain olmadan Authentik'in güvenli çalışabilmesi ve Traefik'in HTTPS sunabilmesi için sertifika üretilmesi gerekir.

> ⚠️ **Not:** Alpine'da `openssl` paketi geçici DNS çözümleme hatası (`ERROR: openssl-3.3.7-r0: temporary error`) verirse, `/etc/resolv.conf` dosyasına geçici olarak okul DNS'ini ekleyin:
> ```bash
> echo "nameserver 194.27.112.201" > /etc/resolv.conf
> echo "nameserver 193.140.245.200" >> /etc/resolv.conf
> ```

Sertifikayı üret:

```bash
apk add openssl
openssl req -x509 -newkey rsa:4096 -nodes \
  -keyout /opt/infra/config/certs/key.pem \
  -out /opt/infra/config/certs/cert.pem \
  -days 3650 \
  -subj "/CN=bmlab.uludag.edu.tr" \
  -addext "subjectAltName=DNS:*.bmlab.uludag.edu.tr,DNS:bmlab.uludag.edu.tr,IP:10.11.0.2,IP:172.18.107.62"
```

Traefik için sertifika tanım dosyasını oluştur:

```yaml
# /opt/infra/config/certs/tls.yml
tls:
  certificates:
    - certFile: /certs/cert.pem
      keyFile: /certs/key.pem
  stores:
    default:
      defaultCertificate:
        certFile: /certs/cert.pem
        keyFile: /certs/key.pem
```

### 4.3 Traefik ForwardAuth Middleware (Authentik Embedded Outpost)

Traefik'in SSO doğrulaması için Authentik'in dâhili (embedded) outpost'unu kullanabilmesi amacıyla `middlewares.yml` dosyasını oluşturun:

```yaml
# /opt/infra/config/certs/middlewares.yml
http:
  middlewares:
    authentik-sso:
      forwardAuth:
        address: "http://authentik-server:9000/outpost.goauthentik.io/auth/traefik"
        trustForwardHeader: true
        authResponseHeaders:
          - X-authentik-username
          - X-authentik-groups
          - X-authentik-email
          - X-authentik-name
          - X-authentik-uid
          - X-authentik-jwt
          - X-authentik-meta-jwks
          - X-authentik-meta-outpost
          - X-authentik-meta-provider
          - X-authentik-meta-app
          - X-authentik-meta-version
```

Bu dosya Traefik tarafından otomatik yüklenecek ve servislerdeki `traefik.http.routers.<isim>.middlewares=authentik-sso@file` etiketiyle çağrılabilecektir.

## 🐳 5. Docker Compose Yapılandırması

Aşağıdaki içeriği `/opt/infra/docker-compose.yml` dosyasına kaydedin. (Authelia yerine PostgreSQL, Redis ve Authentik servisleri eklenmiştir).

```yaml
services:
  dns-server:
    image: technitium/dns-server:latest
    container_name: technitium
    hostname: technitium
    environment:
      - DNS_SERVER_DOMAIN=dns.bmlab.uludag.edu.tr
      - DNS_SERVER_FORWARDERS=194.27.112.201,193.140.245.200
      - DNS_SERVER_RECURSION=AllowOnlyForPrivateNetworks
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.dns.rule=Host(`dns.bmlab.uludag.edu.tr`)"
      - "traefik.http.routers.dns.entrypoints=websecure"
      - "traefik.http.routers.dns.tls=true"
      - "traefik.http.services.dns.loadbalancer.server.port=5380"
      - "traefik.http.routers.dns.middlewares=authentik-sso@file"   
    ports:
      - "0.0.0.0:53:53/udp"
      - "0.0.0.0:53:53/tcp"
      - "0.0.0.0:5380:5380/tcp"
    volumes:
      - ./config/dns:/etc/dns
    restart: always
    networks:
      infra-net:
        ipv4_address: 172.20.0.10

  traefik:
    image: traefik:v3.0
    container_name: traefik
    command:
      - "--api.insecure=true"
      - "--api.dashboard=true"
      - "--providers.docker=true"
      - "--providers.docker.exposedbydefault=false"
      - "--entrypoints.web.address=:80"
      - "--entrypoints.websecure.address=:443"
      - "--entrypoints.ssh.address=:2222"
      - "--providers.file.directory=/certs"
      - "--providers.file.watch=true"
      - "--log.level=DEBUG"
    labels:
      - "traefik.enable=true"
      # Global HTTP to HTTPS redirect
      - "traefik.http.routers.http-catchall.rule=HostRegexp(`{host:.+}`)"
      - "traefik.http.routers.http-catchall.entrypoints=web"
      - "traefik.http.routers.http-catchall.middlewares=redirect-to-https"
      - "traefik.http.middlewares.redirect-to-https.redirectscheme.scheme=https"
      
      # Traefik Dashboard Router
      - "traefik.http.routers.traefik.rule=Host(`traefik.bmlab.uludag.edu.tr`)"
      - "traefik.http.routers.traefik.entrypoints=websecure"
      - "traefik.http.routers.traefik.tls=true"
      - "traefik.http.routers.traefik.service=api@internal"
      - "traefik.http.routers.traefik.middlewares=authentik-sso@file"
    ports:
      - "80:80"
      - "443:443"
      - "8080:8080"
      - "2222:2222"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - ./config/certs:/certs:ro
    dns:
      - 172.20.0.10
    depends_on:
      - dns-server
    restart: always
    networks:
      - infra-net

  postgresql:
    image: postgres:16-alpine
    container_name: authentik-db
    environment:
      POSTGRES_DB: authentik
      POSTGRES_USER: authentik
      POSTGRES_PASSWORD: <AUTHENTIK_DB_PASS>
    volumes:
      - ./data/postgresql:/var/lib/postgresql/data
    restart: always
    networks:
      - infra-net

  redis:
    image: redis:alpine
    container_name: authentik-redis
    restart: always
    networks:
      - infra-net

  authentik-server:
    image: ghcr.io/goauthentik/server:2024.12
    container_name: authentik-server
    command: server
    environment:
      AUTHENTIK_REDIS__HOST: redis
      AUTHENTIK_POSTGRESQL__HOST: postgresql
      AUTHENTIK_POSTGRESQL__NAME: authentik
      AUTHENTIK_POSTGRESQL__USER: authentik
      AUTHENTIK_POSTGRESQL__PASSWORD: <AUTHENTIK_DB_PASS>
      AUTHENTIK_SECRET_KEY: <EN_AZ_50_KARAKTER_RASTGELE_KEY>
      AUTHENTIK_ERROR_REPORTING__ENABLED: "false"
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.authentik.rule=Host(`auth.bmlab.uludag.edu.tr`)"
      - "traefik.http.routers.authentik.entrypoints=websecure"
      - "traefik.http.routers.authentik.tls=true"
      - "traefik.http.services.authentik.loadbalancer.server.port=9000"
    volumes:
      - ./data/authentik/media:/media
      - ./data/authentik/templates:/templates
    depends_on:
      - postgresql
      - redis
    restart: always
    networks:
      - infra-net

  authentik-worker:
    image: ghcr.io/goauthentik/server:2024.12
    container_name: authentik-worker
    command: worker
    environment:
      AUTHENTIK_REDIS__HOST: redis
      AUTHENTIK_POSTGRESQL__HOST: postgresql
      AUTHENTIK_POSTGRESQL__NAME: authentik
      AUTHENTIK_POSTGRESQL__USER: authentik
      AUTHENTIK_POSTGRESQL__PASSWORD: <AUTHENTIK_DB_PASS>
      AUTHENTIK_SECRET_KEY: <EN_AZ_50_KARAKTER_RASTGELE_KEY>
      AUTHENTIK_ERROR_REPORTING__ENABLED: "false"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - ./data/authentik/media:/media
      - ./data/authentik/certs:/certs
      - ./data/authentik/templates:/templates
    depends_on:
      - postgresql
      - redis
    restart: always
    networks:
      - infra-net

networks:
  infra-net:
    driver: bridge
    ipam:
      config:
        - subnet: 172.20.0.0/24
```

> **Not:** `<AUTHENTIK_DB_PASS>` ve `<EN_AZ_50_KARAKTER_RASTGELE_KEY>` değerlerini güvenli şekilde doldurun.

## 🌐 7. Technitium DNS Yapılandırması

### 7.1 Arayüze İlk Erişim

Domain kayıtları aktif olana kadar Technitium arayüzüne **SSH Tüneli** ile erişilebilir:

```bash
ssh -L 15380:10.11.0.2:5380 -N -f core-infra@172.18.107.62
```
Tarayıcıda `http://localhost:15380` adresine gidin.

**Alternatif olarak istemci bilgisayarda (Mac/Windows) `/etc/hosts` düzenlemesi yapılabilir:**

```text
172.18.107.62  dns.bmlab.uludag.edu.tr
172.18.107.62  auth.bmlab.uludag.edu.tr
172.18.107.62  traefik.bmlab.uludag.edu.tr
172.18.107.62  infra.bmlab.uludag.edu.tr
```
*(Bu ayar DNS aktifleştiğinde kaldırılabilir.)*

### 7.2 API Token Alma

1. İlk girişte varsayılan admin şifresini belirleyin.
2. Sağ üstteki kullanıcı menüsünden **Administration → Sessions** sekmesine girin.
3. **Create API Token** diyerek ürettiğiniz token'ı saklayın (`incus-dns-sync` için kullanılacak).

### 7.3 DNS Kayıtları (Zone: `bmlab.uludag.edu.tr`)

Technitium üzerinde `bmlab.uludag.edu.tr` adlı **Primary Zone** oluşturun ve aşağıdaki kayıtları ekleyin:

| Ad | Tür | Değer | Açıklama |
|----|-----|-------|----------|
| `@` | SOA | `dns.bmlab.uludag.edu.tr` | Zone otoritesi |
| `@` | NS | `dns.bmlab.uludag.edu.tr` | Name server |
| `ns1` | A | `10.11.0.2` | DNS sunucusunun adresi |
| `infra` | A | `10.11.0.2` | Traefik giriş noktası |
| `incus` | CNAME | `infra.bmlab.uludag.edu.tr` | Incus Web UI |

## ⚠️ 9. Yaygın Problemler (Troubleshooting)

### ❌ Authentik Gunicorn Sürekli Crash Oluyor
**Hata:** `gunicorn process died, restarting` veya `PermissionError: [Errno 13] Permission denied: '/media/public'`  
**Çözüm:** Authentik container'ı 1000:1000 yetkileri ile çalışır. Klasör izinleri root olarak kalmış demektir. Containerları durdurup klasör izinlerini ayarlayın:
```bash
docker compose down authentik-server authentik-worker
rm -rf /opt/infra/data/authentik
mkdir -p /opt/infra/data/authentik/{media,templates,certs}
chown -R 1000:1000 /opt/infra/data/authentik
chmod -R 755 /opt/infra/data/authentik
docker compose up -d authentik-server authentik-worker
```

### ❌ SSH Tüneli: Port Zaten Kullanımda
**Hata:** `bind [127.0.0.1]:5380: Address already in use`  
**Çözüm:** Farklı lokal port kullanın (`-L 15380:...`) ya da portu tutan işlemi öldürün: `kill $(lsof -t -i:5380)`

### ❌ iptables Kuralı Yanlış Interface'e Yazılmış
**Hata:** İnternetten `curl http://172.18.107.62` yanıt vermiyor.  
**Çözüm:** Doğru interface'i (`enp0s25`) bulup eski kuralları temizleyip yeniden ekleyin (`iptables -t nat -D PREROUTING 1`).

### ❌ DNS Kaydında Typo
**Hata:** `bmailab.uludag.edu.tr` yazılması sonucu sitelerin açılmaması.  
**Çözüm:** Hem `/etc/hosts` hem Technitium CNAME kayıtlarının `bmlab` olarak düzeltildiğinden emin olun.

### ❌ Traefik Dashboard 404 Dönüyor
**Hata:** `/dashboard/` adresi bulunamıyor.  
**Çözüm:** `docker-compose.yml` içinde `--api.dashboard=true` flag'inin ve Traefik için internal service router'ın doğru yazıldığını (`traefik.http.routers.traefik.service=api@internal`) doğrulayın.

### ❌ Alpine'da `apk add openssl` Başarısız
**Hata:** `temporary error (try again later)`  
**Çözüm:** `/etc/resolv.conf`'un `127.0.0.11` göstermesi sebebiyle. Geçici olarak üniversite DNS'ini ekleyin (`echo "nameserver 194.27.112.201" > /etc/resolv.conf`) veya docker ile `alpine/openssl` imajını kullanın.

### ❌ Authentik SSO Sırasında 404 veya JSON Parse Error
**Hata:** SSO girişinde tarayıcının patlaması, `/auth/traefik-forwardauth` 404 hatası vermesi veya uygulamanın yerine yarım JSON dönmesi.  
**Çözüm:** Bu sorun, fazladan kurulan proxy container'larının (`authentik-proxy`) ana sunucuyla çakışmasından kaynaklanır. Yeni mimaride bu proxy kaldırılmıştır. Traefik `forwardAuth` middleware adresinin doğrudan ana sunucuya (Embedded Outpost: `http://authentik-server:9000/outpost.goauthentik.io/auth/traefik`) yönlendirildiğinden (`middlewares.yml`) emin olun.

### ❌ Docker Container'ları Başlamıyor
**Çözüm:** `incus config show infra-gateway | grep security` ile `security.nesting: "true"` olduğunu doğrulayın.
