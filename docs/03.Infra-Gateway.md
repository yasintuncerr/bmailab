# 🛡️ infra-gateway Kurulum Rehberi

## 📌 Genel Bakış

`infra-gateway`, Incus cluster'ının servis katmanını barındıran özel bir container'dır. Alpine Linux tabanlı bu container içinde Docker çalıştırılır; DNS, proxy ve kimlik doğrulama servisleri Docker Compose ile yönetilir.

| Servis     | Image                   | Görev                       |
| ---------- | ----------------------- | --------------------------- |
| Technitium | `technitium/dns-server` | İç DNS yönetimi ve API      |
| Traefik    | `traefik:v3.0`          | HTTP/SSH reverse proxy      |
| Authelia   | `authelia/authelia`     | SSO kimlik doğrulama        |

---

## ⚠️ Ön Koşullar

- Incus cluster kurulu ve çalışır durumda (bkz. `02.Incus-installiation.md`)
- `incusbr0` bridge ağı mevcut
- Host internet erişimi var (Alpine image'ı çekilecek)

---

## 🖥️ 1. Container Oluşturma

### 1.1 Alpine Instance Başlatma

Docker çalıştırabilmek için `security.nesting=true` doğrudan `launch` sırasında verilir; sonradan `restart` gerekmez:

```bash
incus launch images:alpine/3.20 <GATEWAY_INSTANCE_NAME> \
  --profile default \
  -c security.nesting=true
```

| Parametre                 | Açıklama                             | Örnek           |
| ------------------------- | ------------------------------------ | --------------- |
| `<GATEWAY_INSTANCE_NAME>` | Container adı                        | `infra-gateway` |

> 📌 `security.nesting=true` olmadan Docker daemon container içinde başlamaz — `iptables` ve `overlay` filesystem yetkisi bu ayarla gelir.

Container ayağa kalktığını doğrula:

```bash
incus list <GATEWAY_INSTANCE_NAME>
```

`STATUS` sütunu `RUNNING` olmalı.

---

### 1.2 Statik IP Ataması

Container'a sabit IP atamak için:

```bash
incus config device override <GATEWAY_INSTANCE_NAME> eth0 ipv4.address=<GATEWAY_IP>
```

Değişikliğin geçerli olması için container'ı yeniden başlat:

```bash
incus restart <GATEWAY_INSTANCE_NAME>
```

IP'nin atandığını doğrula:

```bash
incus list <GATEWAY_INSTANCE_NAME>
```

`IPV4` sütununda `<GATEWAY_IP>` görünmelidir.

| Parametre      | Açıklama                                               | Örnek          |
| -------------- | ------------------------------------------------------ | -------------- |
| `<GATEWAY_IP>` | infra-gateway'e atanacak sabit IP                      | `10.14.21.20`  |

## 🔥 2. Host UFW Yapılandırması

Docker ve Incus container trafiğinin geçebilmesi için host üzerindeki UFW'nin ilgili arayüzü engellemediğinden emin olunmalıdır.

> 📌 UFW, `FORWARD` chain'ini varsayılan olarak DROP'lar. Bu durum Incus ve Docker container trafiğini sessizce keser. Sorun giderirken ilk kontrol noktası budur.

### 2.1 Incus Bridge Trafiğine İzin Ver

```bash
sudo ufw allow in on <INCUS_BRIDGE_NAME>
sudo ufw allow out on <INCUS_BRIDGE_NAME>
sudo ufw reload
```

Doğrulama:

```bash
sudo ufw status verbose | grep <INCUS_BRIDGE_NAME>
```

ya da geliştirme/test ortamında UFW'yi tamamen devre dışı bırak:

```bash
sudo ufw disable
```

| Parametre             | Açıklama                              | Örnek      |
| --------------------- | ------------------------------------- | ---------- |
| `<INCUS_BRIDGE_NAME>` | Incus köprü arayüzü                   | `incusbr0` |

## 🐋 3. Docker Kurulumu (Alpine)

Container'a bağlan:

```bash
incus exec <GATEWAY_INSTANCE_NAME> -- ash
```

Docker ve Compose'u kur:

```bash
apk update
apk add docker docker-cli-compose curl
```

Docker servisini etkinleştir ve başlat:

```bash
rc-update add docker boot
service docker start
```

Doğrulama:

```bash
docker info | grep "Server Version"
docker compose version
```

Container'dan çık:

```bash
exit
```

## 🐳 5. Docker Compose Yapılandırması

Dosyayı container içinde oluştur:

```bash
incus exec <GATEWAY_INSTANCE_NAME> -- ash
```

```bash
cat > /opt/infra/docker-compose.yml << 'EOF'
services:
  # --- DNS KATMANI ---
  dns-server:
    image: technitium/dns-server:latest
    container_name: technitium
    hostname: technitium
    environment:
      - DNS_SERVER_DOMAIN=<DNS_SERVER_DOMAIN>
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.dns.rule=Host(`<DNS_SUBDOMAIN>.<LAB_DOMAIN>`)"
      - "traefik.http.services.dns.loadbalancer.server.port=<TECHNITIUM_WEB_PORT>"
    ports:
      - "53:53/udp"
      - "53:53/tcp"
    volumes:
      - ./config/dns:/etc/dns
    restart: always

  # --- PROXY KATMANI ---
  traefik:
    image: traefik:v3.0
    container_name: traefik
    command:
      - "--api.insecure=true"
      - "--providers.docker=true"
      - "--providers.docker.exposedbydefault=false"
      - "--entrypoints.web.address=:80"
      - "--entrypoints.ssh.address=:<TRAEFIK_SSH_PORT>"
    ports:
      - "80:80"
      - "<TRAEFIK_DASHBOARD_PORT>:<TRAEFIK_DASHBOARD_PORT>"
      - "<TRAEFIK_SSH_PORT>:<TRAEFIK_SSH_PORT>"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
    restart: always

  # --- GÜVENLİK KATMANI ---
  authelia:
    image: authelia/authelia:latest
    container_name: authelia
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.auth.rule=Host(`<AUTH_SUBDOMAIN>.<LAB_DOMAIN>`)"
      - "traefik.http.services.auth.loadbalancer.server.port=<AUTHELIA_PORT>"
    volumes:
      - ./config/authelia:/config
    restart: always
EOF
```

### Parametre Tablosu

| Parametre                  | Açıklama                                | Örnek Değer                  |
| -------------------------- | --------------------------------------- | ---------------------------- |
| `<LAB_DOMAIN>`             | Laboratuvar iç domain adı               | `bmlab.uludag.edu.tr`        |
| `<DNS_SERVER_DOMAIN>`      | Technitium'un kendi FQDN'i              | `dns.bmlab.uludag.edu.tr`    |
| `<DNS_SUBDOMAIN>`          | Technitium web arayüzü subdomain'i      | `dns`                        |
| `<TECHNITIUM_WEB_PORT>`    | Technitium web arayüzü portu            | `5380`                       |
| `<TRAEFIK_DASHBOARD_PORT>` | Traefik dashboard portu                 | `8080`                       |
| `<TRAEFIK_SSH_PORT>`       | Traefik SSH tünelleme giriş portu       | `2222`                       |
| `<AUTH_SUBDOMAIN>`         | Authelia subdomain'i                    | `auth`                       |
| `<AUTHELIA_PORT>`          | Authelia iç servis portu                | `9091`                       |

## 🌐 7. Technitium DNS Yapılandırması

### 7.0 Arayüze İlk Erişim

DNS zone'ları henüz oluşturulmadığı için `<LAB_DOMAIN>` subdomainleri çözümlenemez. İki yöntemden biri kullanılabilir:

**Yöntem A — SSH Tüneli (önerilen)**

```bash
ssh -L <LOCAL_PORT>:<GATEWAY_IP>:<TECHNITIUM_WEB_PORT> <SSH_USER>@<HOST_WAN_IP>
```

Ardından tarayıcıda:

```
http://localhost:<LOCAL_PORT>
```

**Yöntem B — `/etc/hosts` (istemci makinede geçici)**

```
# macOS / Linux: /etc/hosts
# Windows: C:\Windows\System32\drivers\etc\hosts

<HOST_WAN_IP>  <DNS_SUBDOMAIN>.<LAB_DOMAIN>
<HOST_WAN_IP>  traefik.<LAB_DOMAIN>
<HOST_WAN_IP>  <AUTH_SUBDOMAIN>.<LAB_DOMAIN>
```

> 📌 DNS kayıtları Technitium'dan yayınlanmaya başladıktan sonra bu `/etc/hosts` satırları kaldırılmalıdır.

| Parametre       | Açıklama                                       | Örnek   |
| --------------- | ---------------------------------------------- | ------- |
| `<HOST_WAN_IP>` | core-infra'nın dış (WAN) IP adresi            | —       |
| `<LOCAL_PORT>`  | Yerel makinede kullanılacak port               | `5380`  |
| `<SSH_USER>`    | core-infra SSH kullanıcısı                     | —       |

---

### 7.1 İlk Giriş ve Admin Şifresi

İlk açılışta Technitium, kullanıcı adı ve şifre oluşturmayı ister. Varsayılan kullanıcı adı `admin`'dir.

> ⚠️ Bu şifreyi not al — Technitium API token'ı oluşturmak için gerekli.

---

### 7.2 API Token Alma

DNS kayıtlarını programatik olarak yönetmek için API token'ı gerekir.

1. Technitium arayüzünde sağ üst köşedeki kullanıcı ikonuna tıkla
2. **Administration → Sessions** sekmesine git
3. **Create API Token** butonuna tıkla
4. Token'ı kopyala ve güvenli bir yere kaydet

Bu token `incus-dns-sync` konfigürasyonunda kullanılacak.

---

### 7.3 Zone Oluşturma

1. Sol menüden **Zones** sekmesine git
2. **Add Zone** butonuna tıkla
3. Zone adı: `<LAB_DOMAIN>` (örn: `bmlab.uludag.edu.tr`)
4. Zone tipi: **Primary**
5. **Add** butonuna tıkla

---

### 7.4 Temel DNS Kayıtları

Zone oluşturulduktan sonra aşağıdaki kayıtları ekle:

| #   | Ad                 | Tür   | TTL   | Değer                         | Açıklama                                              |
| --- | ------------------ | ----- | ----- | ----------------------------- | ----------------------------------------------------- |
| 1   | `@`                | SOA   | 15 dk | `<DNS_SERVER_HOSTNAME>.local` | Zone yetkisi                                          |
| 2   | `@`                | NS    | 4 sa  | `<DNS_SERVER_HOSTNAME>.local` | Name server                                           |
| 3   | `ns1`              | A     | 1 sa  | `<GATEWAY_IP>`                | DNS sunucusunun kendi adresi                          |
| 4   | `infra`            | A     | 1 sa  | `<GATEWAY_IP>`                | Traefik'in giriş kapısı (tüm CNAME'ler buraya döner) |
| 5   | `<DNS_SUBDOMAIN>`  | CNAME | 1 sa  | `infra.<LAB_DOMAIN>`          | Technitium web arayüzü erişimi                        |
| 6   | `<AUTH_SUBDOMAIN>` | CNAME | 1 sa  | `infra.<LAB_DOMAIN>`          | Authelia SSO erişimi                                  |
| 7   | `traefik`          | CNAME | 1 sa  | `infra.<LAB_DOMAIN>`          | Traefik dashboard erişimi                             |
| 8   | `*`                | A     | 1 sa  | `<GATEWAY_IP>`                | Wildcard — tanımsız subdomainleri Traefik'e yönlendir |

| Parametre               | Açıklama                                          | Örnek          |
| ----------------------- | ------------------------------------------------- | -------------- |
| `<DNS_SERVER_HOSTNAME>` | Technitium container hostname                     | `technitium`   |

> 📌 Wildcard (`*`) kaydı `incus-dns-sync` tarafından oluşturulan instance kayıtlarının Traefik'e ulaşmasını sağlar. Instance-özel A kayıtları bu wildcard'ın önünde geldiği için çakışma olmaz.

## ⚠️ Yaygın Problemler

### ❌ Docker container'ları başlamıyor

Container'da `security.nesting=true` ayarlanmamış olabilir:

```bash
incus config show <GATEWAY_INSTANCE_NAME> | grep security
```

`security.nesting: "true"` olmalı.

---

### ❌ DNS portu 53 dinlenemiyor

Başka bir işlem portu tutuyor olabilir:

```bash
incus exec <GATEWAY_INSTANCE_NAME> -- netstat -tulpn | grep :53
```

Alpine'da `systemd-resolved` veya başka bir DNS servisi çalışmıyor olması gerekir.

---

### ❌ Traefik container'ları görmüyor

`/var/run/docker.sock` volume mount'u eksik ya da `traefik.enable=true` label'ı unutulmuş olabilir:

```bash
docker inspect <CONTAINER_ADI> | grep traefik.enable
```

---

### ❌ Internet bağlantısı yok (container içinden)

Host UFW, bridge trafiğini engelliyor olabilir. Bkz. Bölüm 2.

Container'ın internet erişimini test et:

```bash
incus exec <GATEWAY_INSTANCE_NAME> -- ping -c 3 8.8.8.8
```

---

### ❌ Technitium API token geçersiz

Yeni token oluşturmak için Technitium arayüzünden **Administration → Sessions** → **Create API Token**.
