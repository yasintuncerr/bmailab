# 🛡️ infra-gateway Kurulum Rehberi

## 📌 Genel Bakış

`infra-gateway`, Incus cluster'ının servis katmanını barındıran özel bir container'dır. Alpine Linux tabanlı bu container içinde Docker çalıştırılır; DNS, proxy ve kimlik doğrulama servisleri Docker Compose ile yönetilir.

| Servis     | Image                   | Görev                  |
| ---------- | ----------------------- | ---------------------- |
| Technitium | `technitium/dns-server` | İç DNS yönetimi ve API |
| Traefik    | `traefik:v3.0`          | HTTP/SSH reverse proxy |
| Authelia   | `authelia/authelia`     | SSO kimlik doğrulama   |

---

## 🖥️ 1. Container Oluşturma

### 1.1 Alpine Instance Başlatma

Docker çalıştırabilmek için `security.nesting=true` doğrudan `launch` sırasında verilir; sonradan `restart` gerekmez:

```bash
incus launch images:alpine/3.20 <GATEWAY_INSTANCE_NAME> \
  --profile default \
  -c security.nesting=true
```

| Parametre                 | Açıklama                             |
| ------------------------- | ------------------------------------ |
| `<GATEWAY_INSTANCE_NAME>` | Container adı (örn: `infra-gateway`) |

> 📌 `security.nesting=true` olmadan Docker daemon container içinde başlamaz — `iptables` ve `overlay` filesystem yetkisi bu ayarla gelir.

---

### 1.3 Statik IP Ataması

Container'a sabit IP atamak için:

```bash
incus config device override <GATEWAY_INSTANCE_NAME> eth0 ipv4.address=<GATEWAY_IP>
```

| Parametre      | Açıklama                                               |
| -------------- | ------------------------------------------------------ |
| `<GATEWAY_IP>` | infra-gateway'e atanacak sabit IP (örn: `10.14.21.20`) |

---

## 🔥 2. Host UFW Yapılandırması

Docker ve Incus container trafiğinin geçebilmesi için host üzerindeki UFW'nin ilgili arayüzü engellemediğinden emin olunmalıdır.

### 2.1 Incus Bridge Trafiğine İzin Ver

```bash
sudo ufw allow in on <INCUS_BRIDGE_NAME>
sudo ufw allow out on <INCUS_BRIDGE_NAME>
```

ya da UFW'yi tamamen devre dışı bırak (geliştirme ortamı için):

```bash
sudo ufw disable
```

| Parametre             | Açıklama                              |
| --------------------- | ------------------------------------- |
| `<INCUS_BRIDGE_NAME>` | Incus köprü arayüzü (örn: `incusbr0`) |

> 📌 UFW, `FORWARD` chain'ini varsayılan olarak DROP'lar. Bu durum Incus ve Docker container trafiğini sessizce keser. Sorun giderirken ilk kontrol noktası budur.

## 🐳 5. Docker Compose Yapılandırması

Dosya: `/opt/infra/docker-compose.yml`

```yaml
services:
  # --- DNS KATMANI ---
  dns-server:
    image: technitium/dns-server:latest
    container_name: technitium
    hostname: technitium
    environment:
      - DNS_SERVER_DOMAIN=<DNS_SERVER_DOMAIN>
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.dns.rule=Host(`<DNS_SUBDOMAIN>.<LAB_DOMAIN>`)"
      - "traefik.http.services.dns.loadbalancer.server.port=<TECHNITIUM_WEB_PORT>"
    ports:
      - "53:53/udp"
      - "53:53/tcp"
    volumes:
      - ./config/dns:/etc/dns
    restart: always

  # --- PROXY KATMANI ---
  traefik:
    image: traefik:v3.0
    container_name: traefik
    command:
      - "--api.insecure=true"
      - "--providers.docker=true"
      - "--providers.docker.exposedbydefault=false"
      - "--entrypoints.web.address=:80"
      - "--entrypoints.ssh.address=:<TRAEFIK_SSH_PORT>"
    ports:
      - "80:80"
      - "<TRAEFIK_DASHBOARD_PORT>:<TRAEFIK_DASHBOARD_PORT>"
      - "<TRAEFIK_SSH_PORT>:<TRAEFIK_SSH_PORT>"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
    restart: always

  # --- GÜVENLİK KATMANI ---
  authelia:
    image: authelia/authelia:latest
    container_name: authelia
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.auth.rule=Host(`<AUTH_SUBDOMAIN>.<LAB_DOMAIN>`)"
      - "traefik.http.services.auth.loadbalancer.server.port=<AUTHELIA_PORT>"
    volumes:
      - ./config/authelia:/config
    restart: always
```

### Parametre Tablosu

| Parametre                  | Açıklama                           | Örnek Değer                 |
| -------------------------- | ---------------------------------- | --------------------------- |
| `<LAB_DOMAIN>`             | Laboratuvar iç domain adı          | `bmailab.uludag.edu.tr`     |
| `<DNS_SERVER_DOMAIN>`      | Technitium'un kendi FQDN'i         | `dns.bmailab.uludag.edu.tr` |
| `<DNS_SUBDOMAIN>`          | Technitium web arayüzü subdomain'i | `dns`                       |
| `<TECHNITIUM_WEB_PORT>`    | Technitium web arayüzü portu       | `5380`                      |
| `<TRAEFIK_DASHBOARD_PORT>` | Traefik dashboard portu            | `8080`                      |
| `<TRAEFIK_SSH_PORT>`       | Traefik SSH tünelleme giriş portu  | `2222`                      |
| `<AUTH_SUBDOMAIN>`         | Authelia subdomain'i               | `auth`                      |
| `<AUTHELIA_PORT>`          | Authelia iç servis portu           | `9091`                      |

## 🌐 7. Technitium DNS Yapılandırması

### 7.0 Uzaktan Erişim (Zone Yapılandırması Öncesi)

DNS zone'ları henüz oluşturulmadığı için `<LAB_DOMAIN>` subdomainleri çözümlenemez. Technitium web arayüzüne iki yöntemden biriyle erişilebilir:

**Yöntem A — `/etc/hosts` (istemci makinede)**

```
# macOS / Linux: /etc/hosts
# Windows: C:\Windows\System32\drivers\etc\hosts

<HOST_WAN_IP>  <DNS_SUBDOMAIN>.<LAB_DOMAIN>
<HOST_WAN_IP>  traefik.<LAB_DOMAIN>
<HOST_WAN_IP>  <AUTH_SUBDOMAIN>.<LAB_DOMAIN>
```

DNS kayıtları Technitium'dan yayınlanmaya başladıktan sonra bu satırlar kaldırılabilir.

**Yöntem B — SSH Tüneli**

```bash
ssh -L <LOCAL_PORT>:<GATEWAY_IP>:<TECHNITIUM_WEB_PORT> <SSH_USER>@<HOST_WAN_IP>
```

Ardından tarayıcıda:

```
http://localhost:<LOCAL_PORT>
```

| Parametre       | Açıklama                                       |
| --------------- | ---------------------------------------------- |
| `<HOST_WAN_IP>` | core-infra'nın dış (WAN) IP adresi             |
| `<LOCAL_PORT>`  | Yerel makinede kullanılacak port (örn: `5380`) |
| `<SSH_USER>`    | core-infra SSH kullanıcısı                     |

---

Servisler ayağa kalktıktan sonra Technitium web arayüzüne erişmek için tarayıcıdan:

```
http://<GATEWAY_IP>:<TECHNITIUM_WEB_PORT>
```

### 7.1 Zone Oluşturma

Arayüzden `<LAB_DOMAIN>` için yetki (authoritative) zone oluştur.

---

### 7.2 Temel DNS Kayıtları

| #   | Ad                 | Tür   | TTL   | Değer                         | Açıklama                                   |
| --- | ------------------ | ----- | ----- | ----------------------------- | ------------------------------------------ |
| 1   | `@`                | SOA   | 15 dk | `<DNS_SERVER_HOSTNAME>.local` | Zone yetkisi                               |
| 2   | `@`                | NS    | 4 sa  | `<DNS_SERVER_HOSTNAME>.local` | Name server                                |
| 3   | `ns1`              | A     | 1 sa  | `<GATEWAY_IP>`                | DNS sunucusunun kendi adresi               |
| 4   | `infra`            | A     | 1 sa  | `<GATEWAY_IP>`                | Traefik'in giriş kapısı                    |
| 5   | `<DNS_SUBDOMAIN>`  | CNAME | 1 sa  | `infra.<LAB_DOMAIN>`          | Technitium web arayüzü erişimi             |
| 6   | `<AUTH_SUBDOMAIN>` | CNAME | 1 sa  | `infra.<LAB_DOMAIN>`          | Authelia SSO erişimi                       |
| 7   | `traefik`          | CNAME | 1 sa  | `infra.<LAB_DOMAIN>`          | Traefik dashboard erişimi                  |
| 8   | `*`                | A     | 1 sa  | `<GATEWAY_IP>`                | Tanımsız subdomainleri Traefik'e yönlendir |

| Parametre               | Açıklama                                          |
| ----------------------- | ------------------------------------------------- |
| `<DNS_SERVER_HOSTNAME>` | Technitium container hostname (örn: `technitium`) |

## ⚠️ Yaygın Problemler

### ❌ Docker container'ları başlamıyor

Container'da `security.nesting=true` ayarlanmamış olabilir:

```bash
incus config show <GATEWAY_INSTANCE_NAME> | grep security
```

---

### ❌ DNS portu 53 dinlenemiyor

Başka bir işlem portu tutuyor olabilir:

```bash
incus exec <GATEWAY_INSTANCE_NAME> -- netstat -tulpn | grep :53
```

---

### ❌ Traefik container'ları görmüyor

`/var/run/docker.sock` volume mount'u eksik ya da `traefik.enable=true` label'ı unutulmuş olabilir.

---

### ❌ Internet bağlantısı yok (container içinden)

Host UFW, bridge trafiğini engelliyor olabilir. Bkz. Bölüm 2.
