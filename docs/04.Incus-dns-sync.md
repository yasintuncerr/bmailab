# 🔁 incus-dns-sync — Otomatik DNS Kayıt Servisi

## 📌 Genel Bakış

`incus-dns-sync`, Incus lifecycle event'lerini dinleyerek her yeni instance için Technitium DNS'e otomatik A kaydı açan, instance silindiğinde ise kaydı kaldıran bir host servisidir.

```
instance-started  →  <instance-adı>.<DNS_ZONE>  →  <IP>   [A kaydı eklenir]
instance-deleted  →  <instance-adı>.<DNS_ZONE>          [A kaydı silinir]
```

Bu sayede kullanıcılar instance'larına doğrudan domain üzerinden erişebilir:

```bash
ssh user@ali-ws01.bmailab.uludag.edu.tr
```

---

## 🧩 Neden Host'ta Çalışır?

`incus monitor`, Incus daemon'ının yalnızca host'ta bulunan Unix socket'ine bağlanır:

```
/var/lib/incus/unix.socket
```

Bu socket, container içinden erişilemez. Bu nedenle servis `infra-gateway` veya başka bir container içinde değil, doğrudan **core-infra host**'unda çalışır.

## ⚙️ Yapılandırma

Dosya: `/etc/incus-dns-sync/incus-dns-sync.conf`

| Parametre          | Açıklama                                            | Örnek                            |
| ------------------ | --------------------------------------------------- | -------------------------------- |
| `TECHNITIUM_API`   | Technitium web API adresi                           | `http://10.14.21.20:5380`        |
| `TECHNITIUM_TOKEN` | API erişim token'ı                                  | `705acf2c...`                    |
| `DNS_ZONE`         | Kayıtların açılacağı zone                           | `bmailab.uludag.edu.tr`          |
| `INSTANCE_IFACE`   | IP'nin okunacağı ağ arayüzü                         | `eth0`                           |
| `IP_WAIT_TIMEOUT`  | IP atanana kadar bekleme süresi (saniye)            | `60`                             |
| `IP_POLL_INTERVAL` | IP kontrol aralığı (saniye)                         | `3`                              |
| `DNS_TTL`          | Oluşturulan A kayıtlarının TTL değeri               | `300`                            |
| `LOG_LEVEL`        | Log ayrıntı seviyesi: `DEBUG` `INFO` `WARN` `ERROR` | `INFO`                           |
| `LOG_FILE`         | Log dosyası konumu                                  | `/var/log/incus-dns-sync/...log` |
| `LOG_MAX_BYTES`    | Rotate eşiği (byte)                                 | `10485760` (10 MB)               |
| `LOG_KEEP_FILES`   | Tutulacak eski log dosyası sayısı                   | `5`                              |

## 🚀 Kurulum

```bash
cd services/incus-dns-sync
sudo bash install.sh
```

Config dosyasını düzenle:

```bash
sudo nano /etc/incus-dns-sync/incus-dns-sync.conf
```

Servisi başlat:

```bash
sudo systemctl start incus-dns-sync
```

## 🛠️ Servis Yönetimi

```bash
# Durum
systemctl status incus-dns-sync

# Başlat / Durdur / Yeniden başlat
systemctl start  incus-dns-sync
systemctl stop   incus-dns-sync
systemctl restart incus-dns-sync

# Canlı log (journald)
journalctl -fu incus-dns-sync

# Log dosyası
tail -f /var/log/incus-dns-sync/incus-dns-sync.log
```

## 🔄 Davranış Detayları

### Başlangıç senkronizasyonu

Servis başladığında tüm `RUNNING` instance'ları tarar ve Technitium'da kaydı olmayan ya da IP'si değişmiş olanları günceller. Bu sayede servis yeniden başlatmalarından sonra tutarsızlık oluşmaz.

### IP bekleme

`instance-started` eventi IP atanmadan önce tetiklenebilir. Script, `IP_WAIT_TIMEOUT` süresince `IP_POLL_INTERVAL` aralıklarla IP'yi kontrol eder. Süre dolduğunda hata loglanır ve kayıt açılmaz.

### Paralel işlem

Her event handler arka planda (`&`) çalışır. Birden fazla instance eş zamanlı başlasa bile event döngüsü bloklanmaz.

### Durdurulan instance'lar

`instance-stopped` eventinde DNS kaydı **korunur** — instance yeniden başlayabilir. Yalnızca `instance-deleted` eventinde kayıt silinir.

### Duplicate kayıt koruması

`dns_add` çağrılmadan önce mevcut kayıt kontrol edilir. IP aynıysa Technitium API'sine gereksiz istek atılmaz.

## 🔍 Sorun Giderme

### Servis başlamıyor

```bash
journalctl -u incus-dns-sync --no-pager -n 50
```

Yaygın nedenler:

- Config dosyası eksik veya hatalı
- `incus` / `curl` / `python3` kurulu değil
- Lock dosyası kalmış: `rm -f /run/incus-dns-sync.lock`

---

### DNS kaydı açılmıyor

Önce Technitium API erişimini doğrula:

```bash
curl -s "http://<TECHNITIUM_API_HOST>:<TECHNITIUM_WEB_PORT>/api/user/session/get?token=<TECHNITIUM_API_TOKEN>"
```

Beklenen yanıt:

```json
{"status":"ok", ...}
```

Log seviyesini `DEBUG` yap ve servisi yeniden başlat:

```bash
# /etc/incus-dns-sync/incus-dns-sync.conf içinde:
LOG_LEVEL="DEBUG"
```

```bash
systemctl restart incus-dns-sync
journalctl -fu incus-dns-sync
```

---

### IP alınamıyor

Instance'ın `eth0` arayüzüne IP atanıp atanmadığını kontrol et:

```bash
incus list <INSTANCE_NAME> --format=json | python3 -c "
import sys, json
d = json.load(sys.stdin)
import pprint; pprint.pprint(d[0].get('state', {}).get('network', {}))
"
```

`INSTANCE_IFACE` config değerinin doğru arayüzü gösterdiğinden emin ol.
