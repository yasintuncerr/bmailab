# 🔁 incus-dns-sync — Otomatik DNS Kayıt Servisi

## 📌 Genel Bakış

`incus-dns-sync`, Incus lifecycle event'lerini dinleyerek her yeni instance için Technitium DNS'e otomatik A kaydı açan, instance silindiğinde ise kaydı kaldıran bir **Go uygulamasıdır**. Host üzerinde systemd servisi olarak çalışır.

```
instance-started  →  <instance-adı>.<DNS_ZONE>  A  <IP>   [kayıt eklenir]
instance-deleted  →  <instance-adı>.<DNS_ZONE>          [kayıt silinir]
```

Bu sayede kullanıcılar instance'larına doğrudan domain üzerinden erişebilir:

```bash
ssh user@ali-ws01.bmlab.uludag.edu.tr
```

---

## 🧩 Neden Host'ta Çalışır?

Servis, Incus daemon'ının yalnızca host'ta bulunan Unix socket'ine bağlanır:

```
/var/lib/incus/unix.socket
```

Bu socket, container içinden erişilemez. Bu nedenle `incus-dns-sync` doğrudan **core-infra host'unda** çalışır — `infra-gateway` veya başka bir container içinde değil.

---

## 🏗️ Mimari

Servis iki katmanlı bir yapı kullanır:

| Katman             | Açıklama |
| ------------------ | -------- |
| **Event Watcher**  | Incus Unix socket'inden tüm projelerdeki lifecycle event'lerini dinler. `instance-started` ve `instance-deleted` geldiğinde reconcile tetikler. Event stream kopması durumunda 5 saniyelik backoff ile yeniden bağlanır. |
| **Reconcile Loop** | Servis başlangıcında ve her `reconciler.interval`'de (varsayılan 5 dk) tam bir diff alır. Event kaçırıldığında veya servis yeniden başlatıldığında garantör olarak çalışır. |

**Desired state:** `dns-enabled` profiline sahip + `Running` durumdaki tüm instance'lar (tüm Incus projeleri taranır)

**Actual state:** Technitium zone'undaki mevcut A kayıtları

**Diff sonucu:**

| Durum | Eylem |
| ----- | ----- |
| Desired'da var, DNS'de yok | `UpsertA` |
| Desired'da var, DNS'de var ama IP farklı | `UpsertA` |
| Desired'da yok, DNS'de var, bu servis tarafından yönetiliyor | `DeleteA` |
| Desired'da yok, DNS'de var, başka servis tarafından oluşturulmuş | Dokunulmaz |

> 📌 Servis, yönettiği kayıtları bellekte `managedNames` kümesi ile takip eder. Bu küme sayesinde başka araçlarla oluşturulan DNS kayıtları asla silinmez.

---

## ⚠️ Ön Koşullar

- core-infra host üzerinde **Go 1.22+** kurulu (yalnızca build için; çalıştırmak için gerekmez)
- Çalışır durumda bir **Incus** sunucusu (`/var/lib/incus/unix.socket` erişilebilir)
- Çalışır durumda bir **Technitium DNS** sunucusu (API token alınmış, bkz. `03.Infra-Gateway.md §7.2`)
- `dns-enabled` Incus profili oluşturulmuş (bkz. `02.Incus-installiation.md §6`)

## 🔨 Build ve Kurulum

Tüm komutlar `services/incus-dns-sync/` dizininde çalıştırılır.

### Build (linux/amd64)

```bash
cd services/incus-dns-sync
make build
```

Binary: `build/incus-dns-sync`

### Cross-compile (örn. macOS'tan Linux ARM64 için)

```bash
make build GOOS=linux GOARCH=arm64
```

### Tam Kurulum (root gerektirir)

```bash
sudo make install
```

Bu komut sırasıyla:

1. Binary'yi `build/incus-dns-sync` olarak derler
2. `/usr/local/bin/incus-dns-sync` olarak kopyalar
3. `/etc/incus-dns-sync/` dizinini oluşturur
4. Örnek config dosyasını `/etc/incus-dns-sync/incus-dns-sync.yaml` olarak kopyalar (varsa üstüne yazmaz)
5. systemd unit dosyasını `/etc/systemd/system/incus-dns-sync.service` olarak kopyalar
6. `systemctl daemon-reload` ve `systemctl enable incus-dns-sync` çalıştırır

## ⚙️ Yapılandırma

Dosya: `/etc/incus-dns-sync/incus-dns-sync.yaml`

```yaml
technitium:
  api_base: "http://<TECHNITIUM_HOST>:<PORT>"   # trailing slash olmadan
  token: "<TECHNITIUM_API_TOKEN>"               # Administration → Sessions
  timeout: 15s

incus:
  socket_path: ""          # boş = varsayılan /var/lib/incus/unix.socket
  dns_profile: "dns-enabled"
  interface: "eth0"
  ip_wait_timeout: 60s
  ip_poll_interval: 3s

dns:
  zone: "bmlab.uludag.edu.tr"
  ttl: 300

reconciler:
  interval: 5m

log:
  level: "info"   # debug | info | warn | error
```

### Parametre Referansı

| Parametre              | Zorunlu | Varsayılan                        | Açıklama |
| ---------------------- | ------- | --------------------------------- | -------- |
| `technitium.api_base`  | ✅       | —                                 | Technitium HTTP API adresi (trailing slash olmadan) |
| `technitium.token`     | ✅       | —                                 | API erişim token'ı |
| `technitium.timeout`   |         | `15s`                             | API çağrıları için HTTP timeout |
| `incus.socket_path`    |         | `/var/lib/incus/unix.socket`      | Incus Unix socket yolu |
| `incus.dns_profile`    |         | `dns-enabled`                     | DNS kaydı açılacak instance'ların taşıması gereken profil adı |
| `incus.interface`      |         | `eth0`                            | IP'nin okunacağı instance ağ arayüzü |
| `incus.ip_wait_timeout`|         | `60s`                             | Instance başladığında IP atanana kadar bekleme süresi |
| `incus.ip_poll_interval`|        | `3s`                              | IP kontrol aralığı |
| `dns.zone`             | ✅       | —                                 | Kayıtların ekleneceği Technitium zone (örn: `bmlab.uludag.edu.tr`) |
| `dns.ttl`              |         | `300`                             | Oluşturulan A kayıtlarının TTL değeri (saniye) |
| `reconciler.interval`  |         | `5m`                              | Periyodik reconcile aralığı |
| `log.level`            |         | `info`                            | Log seviyesi: `debug`, `info`, `warn`, `error` |

## 🚀 Servisi Başlatma

Kurulum ve config düzenlemesi tamamlandıktan sonra:

```bash
# Config dosyasını düzenle
sudo nano /etc/incus-dns-sync/incus-dns-sync.yaml

# Servisi başlat
sudo systemctl start incus-dns-sync

# Başlangıç loglarını izle
journalctl -fu incus-dns-sync
```

Beklenen başlangıç çıktısı:

```
level=INFO msg="incus-dns-sync starting" version=1.0.0 zone=bmlab.uludag.edu.tr profile=dns-enabled
level=INFO msg="Incus connection established"
level=INFO msg="Technitium connection established"
level=INFO msg="reconciler starting" zone=bmlab.uludag.edu.tr interval=5m0s
level=INFO msg="lifecycle event listener started"
level=INFO msg="reconcile completed — no changes"
```

## 🔄 Davranış Detayları

### Başlangıç senkronizasyonu

Servis başladığında hemen bir reconcile çalıştırır. `dns-enabled` profiline sahip tüm `Running` instance'lar taranır. Technitium'da kaydı olmayanlar veya IP'si değişmiş olanlar güncellenir.

### IP bekleme mekanizması

`instance-started` eventi, instance henüz IP almadan önce tetiklenebilir. Bu durumda reconciler IP için `ip_wait_timeout` süresince `ip_poll_interval` aralıklarla polling yapar. IP geldiğinde reconcile tekrar tetiklenir.

### Çoklu proje desteği

Servis, tüm Incus projelerini (`incus project list` eşdeğeri) otomatik olarak tarar. Farklı projelerde aynı adlı iki instance varsa her ikisi de kaydedilir — ancak DNS zone'da çakışma yaşanır. Bu durumdan kaçınmak için instance adları benzersiz tutulmalıdır.

### Durdurulan instance'lar

`instance-stopped` eventinde DNS kaydı **korunur** — instance yeniden başlayabilir. Yalnızca `instance-deleted` eventinde kayıt silinir.

## 🔍 Sorun Giderme

### Servis başlamıyor

```bash
journalctl -u incus-dns-sync --no-pager -n 50
```

Yaygın nedenler:

| Belirti | Olası Neden | Çözüm |
| ------- | ----------- | ----- |
| `config load error` | Config dosyası eksik veya hatalı YAML | `sudo cat /etc/incus-dns-sync/incus-dns-sync.yaml` ile kontrol et |
| `config correction error: technitium.api_base boş olamaz` | Zorunlu alan boş | Config dosyasında `api_base`, `token`, `zone` dolu olmalı |
| `Incus connection error` | Socket erişilemiyor | `ls -la /var/lib/incus/unix.socket` — var mı ve izinler doğru mu? |

---

### DNS kaydı açılmıyor

Önce Technitium API erişimini doğrula:

```bash
curl -s "http://<TECHNITIUM_HOST>:<PORT>/api/user/session/get?token=<TOKEN>"
```

Beklenen: `{"status":"ok",...}`

Daha fazla detay için log seviyesini `debug` yap:

```bash
# /etc/incus-dns-sync/incus-dns-sync.yaml içinde:
#   log:
#     level: "debug"

sudo systemctl restart incus-dns-sync
journalctl -fu incus-dns-sync
```

Debug modunda her reconcile döngüsü, desired ve actual state'i detaylı olarak loglar.

---

### IP alınamıyor

Instance'ın belirtilen interface'te IP aldığını doğrula:

```bash
incus list <INSTANCE_NAME> --format=json | python3 -c "
import sys, json, pprint
d = json.load(sys.stdin)
pprint.pprint(d[0].get('state', {}).get('network', {}))
"
```

Config'deki `incus.interface` değerinin doğru arayüzü gösterdiğinden emin ol.

---

### Instance başlatıldı ama kayıt açılmadı

1. Instance'ın `dns-enabled` profiline sahip olduğunu doğrula:

```bash
incus config show <INSTANCE_NAME> | grep -A5 "profiles:"
```

2. Instance'ın `Running` durumda olduğunu doğrula:

```bash
incus list <INSTANCE_NAME>
```

3. Log'da o instance için reconcile kaydı var mı kontrol et:

```bash
journalctl -u incus-dns-sync --no-pager | grep <INSTANCE_NAME>
```
